🔍 How this analysis works Rule-based static analysis
▶

This tool is a static linter for database schemas — the same way ESLint catches bugs without running your code, this catches design issues without running your SQL. It parses your CREATE TABLE statements into a structured object tree and runs 40+ deterministic rules. No AI, no guessing — every finding either fires or it doesn't.

✅ What it reliably catches
  • Missing primary keys and foreign key constraints
  • Plaintext passwords, exposed PII (SSN, NIN), raw card numbers
  • FLOAT/REAL used for monetary columns
  • 1NF/2NF/3NF normalization violations
  • Missing indexes on FK columns
  • Reserved words as table/column names
  • Query anti-patterns (SELECT *, UPDATE without WHERE)
  • Platform type incompatibilities
⚠️ v1 Limitations
  • No live DB connection — schema-only analysis
  • Cannot see row data, null rates or cardinality
  • No query execution plan or real index stats
  • Cannot validate business logic against constraints
  • Stored procedures, triggers and views not parsed
  • Cross-database FK validation not possible
How to rely on it: Catches the structural, security and naming issues a senior DBA would flag in a design review — before a single line of application code is written. Surfaces 80% of common design mistakes instantly, for free, without sharing your schema with anyone.
